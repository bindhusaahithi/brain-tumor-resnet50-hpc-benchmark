# Reviewer Notes

This repository is structured to show applied science and engineering maturity.

## What To Look At First

1. `README.md` for project framing and reproducibility.
2. `docs/model_card.md` for responsible AI boundaries.
3. `reports/metrics/hardware_results.csv` for benchmark evidence.
4. `scripts/make_visualisations.py` and `images/` for reporting workflow.
5. `app/streamlit_app.py` for live demo readiness.

## Strong Internship Signals

- Clear experimental comparison between CPU and GPU environments.
- Same model setup used across environments.
- Results stored as reusable metrics rather than only screenshots.
- Model limitations and clinical safety caveats are explicit.
- Project has source-code separation, tests, linting, and CI.
